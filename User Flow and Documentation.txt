================================================================================
  TULIO OS — PRODUCT DOCUMENTATION
  Interactive Desktop Experience v2.0
  tuliocareli.com
================================================================================

  "Tulio OS é um projeto de portfólio interativo que simula
   um sistema operacional Windows XP dentro do navegador.
   O objetivo é demonstrar domínio técnico em front-end,
   engenharia de APIs e design de experiência de produto —
   ao mesmo tempo em que funciona como um retrato pessoal."

                                                         — Túlio Careli


================================================================================
  1. PRODUCT OVERVIEW
================================================================================

  TIPO:        Web-based Interactive Experience / Portfolio
  PLATAFORMA:  Browser (Desktop & Mobile)
  DOMÍNIO:     tuliocareli.com
  STACK:       HTML5 + CSS3 + Vanilla JS (ES6+) | Vercel Serverless
  PÚBLICO:     Recrutadores, PMs, Designers, Devs, curiosos

  PROPOSTA DE VALOR:
  Portfólio disfarçado de sistema operacional nostálgico (Windows XP),
  onde cada "aplicativo" é uma demonstração técnica ou pessoal.
  O visitante não navega páginas — ele usa um OS.

  4.3 TULIO EARTH (3D & Street View)
  ─────────────────────────────────────────────────────
  Flow completo:
  1. Abertura: CesiumJS inicializa globo 3D com token Ion (via /api/cesium)
  2. Navegação: voo livre pelo mundo, toggle relevo 3D, atmosfera
  3. Street View:
     a. Click em [Street View Aqui!]
     b. Raycasting do centro da câmera → colisão com globo
     c. Coordenadas Lat/Lng extraídas
     d. Consulta API Mapillary V4 (raio 50m via /api/mapillary)
     e. MapillaryJS renderiza overlay 360° sobre o globo
     f. "Sair" volta ao modo globo


  ─────────────────────────────────────────────────────
  4.4 DOOM.EXE (Emulação x86 no Browser)
  ─────────────────────────────────────────────────────
  1. Duplo clique em doom.exe
  2. Carrega js-dos.js (DOSBox via WASM) de /libs/js-dos
  3. Monta bundle doom.jsdos no filesystem virtual
  4. Audio via Web Audio API + captura teclado/mouse
  5. Controles: Setas=Mover, CTRL=Atirar, Space=Portas
  6. onClose: ci.exit() para liberar CPU/memória

================================================================================
  5. WINDOW MANAGEMENT SYSTEM
================================================================================

  Comportamento padrão de todas as janelas:
  • Arrastáveis (mouse + touch via drag handlers manuais)
  • Minimizar → via Taskbar (toggle xp-win--minimized)
  • Maximizar → preenche xpDesktopArea (restore via toggle)
  • Fechar (X) → remove do DOM, executa onClose callback
  • Múltiplas janelas abertas simultaneamente
  • Z-index dinâmico (variável zTop incrementada a cada foco)
  • Resize em 8 direções (n/ne/e/se/s/sw/w/nw)
  • Aspect ratio lock para: earth, doom, keen, paint
  • SFX: click no abrir, error em falhas

  6. TECH STACK
================================================================================

  FRONTEND
  ────────
  Runtime:     Vanilla HTML5 + CSS3 + JavaScript (ES6+)
  Arquivos:
  • index.html    — Entry point, Fake BIOS
  • desktop.js    — Engine do Tulio OS (5162 linhas, IIFE)
  • script.js     — Widgets da landing page (inativo no flow atual)
  • style.css     — Design system base, CSS Variables, tema neon
  • desktop.css   — Estilos do OS (janelas, taskbar, ícones)
  • apps.css      — Estilos internos de apps (TC Explorer, Orkut, etc.)
  • thps.css      — Score widget THPS (inativo no flow atual)

  Técnicas-chave:
  • Canvas API (visualizadores, animações boot)
  • Drag & Drop manual (sem libs externas)
  • requestAnimationFrame (animações contínuas)
  • localStorage (scraps Orkut, posição de ícones)
  • CSS Custom Properties (design tokens, tema retrô)
  • Web Audio API (SFX do OS)
  • WebAssembly (js-dos para emulação x86)
  • Raycasting de câmera (CesiumJS → MapillaryJS)


  BACKEND / DEPLOY
  ────────────────
  Plataforma:  Vercel (Serverless Functions)
  Runtime:     Node.js 24.x
  Endpoints:
  • POST /api/chat     — Chat Gemini AI (Tulio Messenger)
  • GET  /api/cesium    — Proxy p/ Cesium Ion token
  • GET  /api/mapillary — Proxy p/ Mapillary token

  CORS:
  • https://tuliocareli.com
  • http://localhost:3000
  • http://127.0.0.1:5500
  • *.dominiodotulio.vercel.app (regex preview deploys)

================================================================================
  9. DESIGN DECISIONS & TRADE-OFFS
================================================================================

  1. VANILLA JS (sem frameworks)
     ✅ Zero build step, deploy direto na Vercel
     ✅ Performance máxima (sem virtual DOM overhead)
     ⚠️ Arquivo desktop.js com ~5K linhas — monolítico

  2. IIFE PATTERN (desktop.js)
     ✅ Encapsulamento total, zero poluição de namespace
     ✅ window.tcDesktop como API pública controlada
     ⚠️ Debugging mais difícil sem source maps

  3. LAZY MOUNTING
     ✅ OS só renderiza quando usuário interage
     ✅ YouTube SDK carregado sob demanda

  4. TOKENS VIA SERVERLESS PROXY
     ✅ Chaves nunca expostas ao client
     ✅ CORS restrito a domínios autorizados

  5. FLASH VIA RUFFLE
     ✅ Experiência nostálgica preservada
     ⚠️ AS3 sem servidor original = renderização parcial


================================================================================
  10. ACCESSIBILITY & PERFORMANCE NOTES
================================================================================

  • Todos os ícones possuem labels de texto (não só emoji)
  • Janelas navegáveis via touch em mobile
  • SFX respeitam autoplay policy (catch silencioso)
  • CesiumJS e MapillaryJS carregados eager (pesado, ~5MB)
    → Trade-off: necessário para Earth funcionar sem lag
  • Doom cleanup via ci.exit() previne memory leaks
