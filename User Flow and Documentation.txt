================================================================================
  3. TECNOLOGIAS UTILIZADAS
================================================================================

  FRONTEND
  ────────
  Linguagem:   HTML5 + CSS3 + JavaScript (Vanilla ES6+)
  Arquivo CSS: style.css, desktop.css, apps.css, thps.css
  Arquivos JS: desktop.js (OS completo), script.js (página principal)

  Principais técnicas:
  - Canvas API (animações de intro: starfield + matrix + header rain)
  - Drag & Drop manual (janelas do OS, sem libs externas)
  - requestAnimationFrame (animadores de canvas e visualizador)
  - localStorage (scraps do Orkut persistidos no browser)
  - Z-index dinâmico (gerenciamento de foco de janelas)
  - Micro-interações e SFX com Web Audio API (arquivos .mp3 em /sons)
  - CSS Custom Properties (sistema de design, tema neon/retrô)
  - CSS Animations (glitch loop, domain rings, THPS score, etc.)


  BACKEND / DEPLOY
  ────────────────
  Plataforma:  Vercel (Serverless Functions)
  Runtime:     Node.js 24.x
  Entry:       api/chat.js (Vercel Serverless Function)

  CORS configurado para:
  - https://tuliocareli.com
  - http://localhost:3000
  - http://127.0.0.1:5500
  - *.dominiodotulio.vercel.app (regex preview deploys)


  ANALYTICS
  ─────────
  Ferramenta:  Vercel Web Analytics
  Método:      Snippet vanilla JS (/_vercel/insights/script.js)
  Dados:       Pageviews, referral, Web Vitals — sem cookies.

  4. APIs E INTEGRAÇÕES
================================================================================

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  GOOGLE GEMINI API                                                       │
  │  Propósito:   Alimenta o "Túlio AI" no app Tulio Messenger (MSN-style)  │
  │  Pacote:      @google/generative-ai ^0.21.0                             │
  │  Modelo:      gemini-flash-latest                                        │
  │  Endpoint:    POST /api/chat                                             │
  │  Auth:        GEMINI_API_KEY (env var na Vercel, nunca exposta)          │
  │  System Prompt: personagem "Túlio", designer de BH, digitando como      │
  │                 se fosse MSN 2005, com travas contra jailbreak           │
  │  Limitações:  Respostas curtas (máx 3 frases), assuntos restritos       │
  └─────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  YOUTUBE IFRAME API                                                       │
  │  Propósito:   Reprodução de vídeos do canal @Fonzarelli23 dentro do      │
  │               TC Explorer (aba "YouTube 2007")                           │
  │  Carregamento: Lazy — script injetado no DOM só quando a aba é aberta    │
  │  SDK:         https://www.youtube.com/iframe_api                         │
  │  Auth:        Pública (sem API key — uso apenas de iframe embed)         │
  └─────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  WAYBACK MACHINE / INTERNET ARCHIVE                                       │
  │  Propósito:   Carregar versões históricas de sites dentro do TC Explorer  │
  │  Sites:       Tony Hawk Underground (2003), NFS Underground 2 (2004),    │
  │               Cartoon Network BR (2004)                                  │
  │  Método:      iframe apontando para URLs do web.archive.org              │
  │  Auth:        Pública (sem autenticação)                                 │
  └─────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  RUFFLE (Flash Player Emulator)                                           │
  │  Propósito:   Emular e rodar arquivos .SWF (Flash) no browser moderno   │
  │  Apps usados: BuddyPoke (Orkut), TG Motocross 2, BMX Stunts,            │
  │               Create a Ride, Doom (versão flash)                        │
  │  Carregamento: https://unpkg.com/@ruffle-rs/ruffle                      │
  │  Auth:        CDN pública, sem autenticação                              │
  │  Limitação:   Texturas ActionScript 3 sem servidor original → alguns     │
  │               SWFs carregam parcialmente ("modo fantasma")               │
  └─────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  SPOTIFY WIDGET (MOCK)                                                    │
  │  Propósito:   Exibir "Now Playing" na sidebar                            │
  │  Método:      Rotação automática de faixas hardcoded (sem API real)      │
  │  Nota:        Widget de aparência mas sem integração OAuth com Spotify.  │
  │               Futuras versões poderiam usar Spotify Web API + token.     │
  └─────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  LOCALSTORAGE                                                             │
  │  Propósito:   Persistir "scraps" (recados) do Orkut simulado            │
  │  Chave:       "orkut_scraps"                                             │
  │  Formato:     JSON Array → [{ text: string, date: string }]             │
  │  Escopo:      Browser do usuário — limpar cache apaga os scraps          │
  └─────────────────────────────────────────────────────────────────────────┘
